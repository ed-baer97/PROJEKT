# Описание границ проекта (Scope Statement)

## Проект: платформа обучения и адаптации проектных менеджеров на базе ИИ

**Цель проекта:** новый проектный менеджер самостоятельно осваивает материалы и подтверждает знания тестами, без постоянного участия наставника.

**Продукт проекта:** платформа с собственной базой знаний, в которую вручную загружаются существующие материалы компании (Word-документы), с тремя ИИ-функциями:

1. генерация обучающих презентаций по документам;
2. генерация тестов по документам;
3. чат-бот, отвечающий по загруженной информации.

Связка онбординга: изучил презентацию → прошёл тест → в процессе работы спрашиваешь бота.

---

## Входит в границы проекта

| № | Работа | Описание |
|---|--------|----------|
| 1 | **Оценка материалов** | Сколько Word-документов, как они структурированы, разбивка по разделам. Первая задача проекта |
| 2 | **Модуль загрузки контента** | Админ-панель для ручной загрузки Word-файлов в базу проекта с раскладкой по разделам |
| 3 | **Обработка материалов** | Извлечение текста из Word-документов, индексация контента для ИИ |
| 4 | **ИИ-презентации** | Генерация обучающей презентации по документу или разделу: слайды с ключевыми тезисами, по которым новичок проходит обучение |
| 5 | **ИИ-тесты** | Генерация тестовых вопросов по выбранному разделу базы, прохождение теста сотрудником, сохранение и просмотр результатов |
| 6 | **Чат-бот** | Сотрудник задаёт вопрос своими словами → получает ответ, составленный по загруженным материалам, со ссылкой на документ-источник. На вопросы вне базы ИИ отвечает «в базе этого нет», а не выдумывает |
| 7 | **Тестовая эксплуатация** | Прогон на пилотной группе, исправление замечаний |
| 8 | **Внедрение** | Запуск рабочей версии и выдача доступа всем проектным менеджерам |

## Не входит в границы проекта

- **«Разведчик» тендеров, ИИ-анализ смет, аналитика закупок** — фаза 2, отдельный проект после запуска обучающего контура (см. раздел «Фаза 2» ниже).
- Работа с видеоматериалами (хранение, транскрибация) — только Word-документы.
- Создание и редактирование обучающего контента вручную (презентации и тесты генерирует ИИ по существующим документам).
- Ручное написание банка тестовых вопросов.
- Интеграции с HR / кадровыми системами.
- Мобильное приложение.
- Обучение других ролей, кроме проектных менеджеров.

---

## Допущения

- Обучающие материалы (Word-документы) существуют и готовы к загрузке.
- Назначен ответственный за загрузку и актуализацию материалов (роль в проекте, не задача разработки).
- Есть пилотная группа ПМ для тестовой эксплуатации.
- Качество сгенерированных презентаций и тестов выборочно проверяет назначенный эксперт, прежде чем они показываются новичкам (роль в проекте, не задача разработки).

## Ограничения

- Платформа реализуется только как веб-приложение (мобильное приложение не разрабатывается).
- Сроки исходного плана взяты ориентировочно и подлежат пересмотру после оценки материалов.
- Работа только с существующими материалами — новый контент в рамках проекта вручную не создаётся.

## Главный открытый риск

Качество и структура исходных Word-документов пока неизвестны, а от них напрямую зависит качество генерируемых презентаций и тестов: из плохо структурированных документов получатся плохие учебные материалы. Поэтому **первой задачей проекта стоит оценка материалов** — короткая работа (несколько дней), без которой любые оценки сроков остаются условными.

---

## Критерии приёмки

> Цифры ориентировочные — подставить свои после согласования.

- Бот корректно отвечает не менее чем на **80%** контрольного списка типовых вопросов новичка.
- По каждому разделу базы генерируется обучающая презентация; эксперт принимает не менее **80%** презентаций без серьёзных правок.
- По каждому разделу базы генерируется тест; эксперт признаёт годными не менее **80%** вопросов.
- Пилотная группа из **N** человек прошла онбординг на платформе.
- Доступ выдан всем действующим ПМ группы компаний.

---

## Укрупнённый план работ (фаза 1)

1. Оценка материалов (количество и структура Word-документов)
2. Модуль загрузки контента (админ-панель)
3. Обработка материалов: извлечение текста из Word, индексация
4. ИИ-генерация обучающих презентаций
5. ИИ-генерация тестов
6. Чат-бот по загруженной информации
7. Тестовая эксплуатация на пилотной группе
8. Внедрение рабочей версии

---

## Фаза 2 (отдельный проект, после запуска фазы 1)

| Модуль | Описание | Статус |
|--------|----------|--------|
| **«Разведчик» тендеров** | Мониторинг фиксированного списка источников (ЕИС, реестр гос.экспертизы и т.п., список пополняемый) по заданным тегам/ключевым словам. Ищет годовые планы закупок и объекты на гос.экспертизе — то есть будущие тендеры до их объявления | В составе фазы 2 |
| **ИИ-анализ смет** | Готовые сметы загружаются в систему в виде Excel-файлов, ИИ их анализирует и проверяет | В составе фазы 2 |
| **Аналитика закупок по группе компаний** | Дашборд по данным закупок всех компаний группы | Под вопросом — возможное расширение, решение позже |

**Границы фазы 2 (предварительно):**

- Разведчик ищет только по согласованному списку источников — «весь интернет» не входит.
- ИИ-анализ смет работает с загруженными Excel-файлами; составление смет «с нуля» не входит.
- Результат анализа смет — рекомендация для специалиста, юридически значимые сметы модуль не заменяет.
- Детальный устав фазы 2 составляется после запуска фазы 1.

**Подготовка к фазе 2 (сделать заранее, во время фазы 1):**

- Собрать 3–5 реальных смет-примеров в Excel от разных компаний группы. По ним станет ясно, насколько единообразен формат.

**Риск фазы 2:** если каждая компания группы ведёт сметы в своём формате Excel, разбор разнородных файлов станет самой сложной частью ИИ-анализа смет. Оценивать модуль без смет-примеров нельзя.
